Zyxel switch has no provider. Monorepo structure not yet established for cloud-init, quadlets, compose, and env layers.

## Constraints

- Not connected to live infra yet — must apply cleanly offline
- Secrets never stored in repo — only secret manager references
- Zyxel switch has no Tofu provider — document via comments only
- Each Proxmox node is standalone (not clustered) — requires aliased provider per node

## Optimization Targets

- All Proxmox-manageable resources codified (node1, node3, node4 — VMs, LXCs, networking, pools)
- OPNsense router1 modeled via browningluke/opnsense provider
- Import blocks present and ready for when live API access is wired up

## Observation Indicators

- tofu plan exit code (must be 0)
- No secrets or credentials appear in any tracked file

## Acceptance

tofu plan runs clean across all four targets (node1, node3, node4, opnsense/router1). Import blocks exist in imports.tf files, commented out. Monorepo directory structure established for cloud-init, quadlets, compose, and env layers.
