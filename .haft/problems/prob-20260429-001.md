# Complete declarative homelab IaC baseline

## Signal

Existing .tf files partially model node3+node4 Proxmox VMs and networking. node1 (3rd hypervisor), all LXCs, and router1 (OPNsense) are unmodeled. Zyxel XMG1915-10E switch has no Tofu provider but supports SSH/CLI management via Ansible. No resources imported into state. Monorepo structure not yet established for cloud-init, quadlets, compose, env, and Ansible layers.

## Constraints

- Not connected to live infra yet — must apply cleanly offline
- Secrets never stored in repo — only secret manager references
- Zyxel requires Ansible via SSH/CLI — no Tofu provider
- Each Proxmox node is standalone (not clustered) — requires aliased provider per node
- No Nebula cloud dependency for switch management

## Optimization Targets

- All Proxmox-manageable resources codified (node1, node3, node4 — VMs, LXCs, networking, pools)
- OPNsense router1 modeled via browningluke/opnsense Tofu provider
- Zyxel XMG1915-10E VLAN and port config managed via Ansible SSH playbooks

## Observation Indicators

- tofu plan exit code (must be 0 across all workspaces)
- ansible --check exits clean
- No secrets or credentials appear in any tracked file

## Acceptance

tofu plan runs clean across node1/node3/node4/router1 workspaces. Ansible playbooks cover Zyxel VLAN and port config and pass --check. Import blocks present and commented. Monorepo directory structure established for cloud-init, quadlets, compose, env, and ansible layers.

## Blast Radius

This repo only. No live infra touched.

## Reversibility

high
